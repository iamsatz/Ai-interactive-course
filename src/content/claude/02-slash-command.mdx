<ConceptSection id="slash-command" term="Slash Command" category="claude" oneLiner="How you trigger a skill. Shortcuts for your entire process.">

A **slash command** is how you invoke a skill by name. Type `/start` and Claude executes the "start a new session" skill. Type `/figma-to-code` and it runs your Figma import workflow. The slash is just the trigger — the command name matches a skill file.

Slash commands turn multi-step workflows into single keystrokes. The same way keyboard shortcuts in any tool let you execute complex operations with muscle memory, slash commands let you run entire Claude workflows without writing instructions from scratch.

<Diagram />

You can browse available slash commands by typing `/` in Claude Code — a dropdown shows all installed skills. Yours appear alongside the built-in ones.

<Analogy>/start, /figma-to-code — keyboard shortcuts, but each one executes a full multi-step workflow, not just one action. The command is the button; the skill is what runs when you press it.</Analogy>

<WhyItMatters accent="var(--accent-claude)">The difference between Claude as a chatbot and Claude as a keyboard-shortcut system. Chatbots need instructions each time. Slash commands make your whole process available on demand — no re-explaining, no context-setting, just invoke and it runs.</WhyItMatters>
